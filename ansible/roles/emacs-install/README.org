** Role Variables

- =requested_emacs_version= (default ="="): build only the matching entries
  from =emacs_versions=; empty builds them all. A bare version (e.g. =master=)
  selects every variant with that version; a =version-suffix= name (e.g.
  =master-lucid=, =30.2-gtk3=) selects just that one variant. Set it on the
  command line via =-e emacs_version_name==.
- =emacs_git_repo_url= (default =https://github.com/emacs-mirror/emacs.git=):
  git remote used for =source: git= Emacs builds. The GitHub mirror is the
  default for resilience. Override it explicitly, for example with
  =-e emacs_git_repo_url=https://git.savannah.gnu.org/git/emacs.git=, if you
  want to build from Savannah instead.
- =emacs_versions= (vars/main.yaml): the list of Emacs builds (master + the
  configured releases) with their =configure= flags and tree-sitter version.
- =emacs_keep_daily_builds= (default =3=): the maximum number of
  date-stamped daily =master= build days to keep under =~/.local=. If fewer
  than that exist, keep whatever is available. See "Daily master builds"
  below.

** Daily master builds

The =master= build installs to =~/.local/emacs-master-YYYY-MM-DD[-suffix]= and
is skipped when today's directory already exists, so it is effectively one
build per day. After each run the role prunes these date-stamped directories
down to the newest =emacs_keep_daily_builds= *at most*; on a fresh system that
has only built today, that means keeping just today's build. The undated
legacy =emacs-master-<suffix>= is never touched.

To build one automatically every day, two scripts live in
=infra/ansible/scripts/=:

- =o-emacs-daily-cron [install|remove|show]=: installs a managed crontab block
  that builds the master daily at noon (=EMACS_DAILY_HOUR=, default 12) and
  adds an =@reboot= catch-up entry.
- =o-emacs-daily-build [REPO]=: the worker the crontab calls. It is idempotent
  (skips when today's build exists) and only builds once the scheduled hour has
  passed, so a boot before noon defers to the timed entry while a boot after
  noon builds the daily that was missed while the machine was off.

A stock crontab does not run jobs missed while powered off; the =@reboot= entry
plus this gating is what provides the catch-up. It covers power-off but not
suspend/resume — for that, prefer a systemd user timer with =Persistent=true=.
The build runs =make install= under ansible =become=, so unattended cron runs
need passwordless sudo.

** Build metadata

Each install directory also gets a =neo/buildinfo.md= file (a peer of
=bin/=) recording the =vars/main.yaml= entry that drove that specific
build/variant, plus a trimmed =./configure= summary annotated with the
detected version of each linked library (via
=scripts/summarize-configure=), for later debugging or auditing which
flags, libraries, and tree-sitter version a given install was built
with.
